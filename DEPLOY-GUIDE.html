BEFORE YOU START

Prerequisites

You need three free accounts and one piece of software. Everything below is free.

1
Node.js Required
Download from nodejs.org — install the LTS version. After installing, open Terminal and run node --version — you should see v18 or higher.
2
GitHub account Required
Create a free account at github.com. This stores your code and connects to Vercel for auto-deployment.
3
Supabase account Required
Create a free account at supabase.com. This is your database and authentication backend.
4
Vercel account Required
Create a free account at vercel.com — sign up with your GitHub account so they are linked.
STEP 1

Set Up Supabase

1.1 — Create a new project

→
Go to supabase.com → click Sign In → sign in with GitHub
→
Click New project in your dashboard
→
Fill in: Project name = hoops-pro (or anything), Database Password = create a strong one and save it somewhere, Region = pick the closest to your location
→
Click Create new project — wait about 2 minutes for it to finish setting up

1.2 — Run the database SQL

This creates all the tables your app needs with security rules so each user only sees their own data.

→
In your Supabase project, look at the left sidebar → click SQL Editor
→
Click New query
→
Open the file supabase-setup.sql from your project folder in any text editor, copy all the contents
→
Paste it into the SQL Editor → click Run
✅
You should see: "Success. No rows returned" — this means all tables were created correctly.

1.3 — Get your API keys

You need two values from Supabase to connect your app to the database.

→
In Supabase sidebar → click the gear icon (Project Settings) → click API
→
Copy Project URL — looks like: https://abcdefghijkl.supabase.co
→
Copy anon public key — a very long string starting with eyJhbGci...
⚠️
Keep these safe. Save them in a text file on your computer. You'll use them in Step 2 and Step 4.

1.4 — Enable Google Sign-In Optional

Skip this if you only want email & password login.

→
Go to console.cloud.google.com → create or select a project → go to APIs & Services → Credentials → click Create Credentials → OAuth 2.0 Client IDs
→
Application type = Web application. Under Authorized redirect URIs add: https://YOUR_PROJECT_ID.supabase.co/auth/v1/callback
→
Copy the Client ID and Client Secret
→
Back in Supabase → Authentication → Providers → Google → toggle ON → paste Client ID and Client Secret → Save
STEP 2

Set Up Locally

2.1 — Extract the project

Download and extract the hoops-pro-typescript.zip file. You'll get a folder called hoops-ts. Put it somewhere easy to find like your Desktop or Documents.

2.2 — Install dependencies

Open your Terminal (Mac/Linux) or Command Prompt / PowerShell (Windows). Navigate to your project folder:

cd path/to/hoops-ts
# Example on Mac:   cd ~/Desktop/hoops-ts
# Example on Windows: cd C:\Users\YourName\Desktop\hoops-ts

Then install all the packages Next.js needs:

npm install

This takes 1–2 minutes and creates a node_modules folder.

2.3 — Add your Supabase keys

→
In the hoops-ts folder, find the file called .env.local.example
→
Copy that file and rename the copy to exactly .env.local (note the dot at the start)
→
Open .env.local in any text editor (Notepad, TextEdit, VS Code) and replace the placeholder values with your real keys from Step 1.3:
NEXT_PUBLIC_SUPABASE_URL=https://your-actual-project-id.supabase.co
NEXT_PUBLIC_SUPABASE_ANON_KEY=your-actual-very-long-anon-key-here
🔒
Never share or commit .env.local — it contains your secret keys. It's already in .gitignore so Git will automatically ignore it.

2.4 — Test it locally (recommended)

npm run dev

Open http://localhost:3000 in your browser. You should see the login page. If it works locally, it will work on Vercel too.

Press Ctrl+C in Terminal to stop the local server when done.

STEP 3

Push to GitHub

3.1 — Install Git (if you don't have it)

  • Mac: Run git --version in Terminal — if not installed, it will prompt you to install it, or install via Homebrew: brew install git
  • Windows: Download from git-scm.com and install it
  • Linux: Run sudo apt install git

3.2 — Create a GitHub repository

→
Go to github.com/new
→
Repository name: hoops-pro
→
Set visibility to Public (required for free Vercel deployment)
→
Do NOT check "Add a README file" — your project already has one
→
Click Create repository

3.3 — Push your code

In Terminal, inside your hoops-ts folder, run these commands one by one:

# 1. Initialize Git
git init

# 2. Stage all files
git add .

# 3. Create your first commit
git commit -m "Initial commit: Hoops Pro"

# 4. Connect to your GitHub repo
#    Replace YOUR_USERNAME with your actual GitHub username
git remote add origin https://github.com/YOUR_USERNAME/hoops-pro.git

# 5. Push to GitHub
git branch -M main
git push -u origin main
💡
GitHub may ask you to sign in. Use your GitHub username and password. If you have 2FA enabled, you need to use a Personal Access Token instead of your password — create one at github.com/settings/tokens with repo scope.

After running these commands, refresh your GitHub repo page — you should see all your files listed there.

STEP 4

Deploy on Vercel

4.1 — Import your repository

→
Go to vercel.com/new
→
Click Continue with GitHub and authorise Vercel to access your repos
→
Find hoops-pro in the list → click Import

4.2 — Add environment variables Critical

Before clicking Deploy, expand the Environment Variables section and add both of these:

NameValue
NEXT_PUBLIC_SUPABASE_URLYour Supabase Project URL (from Step 1.3)
NEXT_PUBLIC_SUPABASE_ANON_KEYYour Supabase anon public key (from Step 1.3)
🚨
If you skip this step, the app will deploy but auth will not work. You must add the environment variables before deploying.

4.3 — Deploy

→
Click Deploy — Vercel builds and deploys in about 60 seconds
→
You'll get a live URL like: https://hoops-pro-abc123.vercel.app — copy this URL

4.4 — Set Supabase redirect URLs Critical

Supabase needs to know your live domain so auth redirects work correctly.

→
Go to Supabase → Authentication → URL Configuration
→
Set Site URL to your Vercel URL: https://hoops-pro-abc123.vercel.app
→
Under Redirect URLs, click Add URL and add: https://hoops-pro-abc123.vercel.app/**
→
Click Save
🎉
Your app is now live! Visit your Vercel URL and create an account to start using Hoops Pro.

ONGOING

Pushing Future Updates

Every time you make changes to your code locally and want to push them live, run these three commands in Terminal from your hoops-ts folder:

git add .
git commit -m "Describe what you changed"
git push

Vercel automatically detects the push and rebuilds your app in about 30 seconds. No further action needed.

HELP

Troubleshooting

'next' is not recognized as a command
You haven't installed dependencies yet. Run npm install first, then npm run dev.
App keeps redirecting back to /login after signing in
Your Supabase redirect URL is not set. Go to Supabase → Authentication → URL Configuration and add your Vercel domain as both the Site URL and a Redirect URL (with /** at the end).
Build fails on Vercel with "supabase" errors
Environment variables were not added. Go to your Vercel project → Settings → Environment Variables and add NEXT_PUBLIC_SUPABASE_URL and NEXT_PUBLIC_SUPABASE_ANON_KEY, then redeploy.
Viewer / Overlay shows old data or doesn't update
The viewer and overlay sync via browser localStorage. They must be open in the same browser on the same device as the scoreboard admin page. Different devices or browsers won't share live data without additional Supabase real-time setup.
git push asks for password and fails
GitHub no longer accepts passwords for git operations. Create a Personal Access Token at github.com/settings/tokens (select repo scope), then use that token as your password when prompted.
Port 3000 is already in use
Something else is using port 3000. Run npm run dev -- -p 3001 to use port 3001 instead, then open localhost:3001.
Logo doesn't appear in Viewer or Overlay
Logos are stored in browser localStorage as base64 images. Upload the logo in the Scoreboard admin page, then open Viewer/Overlay in the same browser — they refresh every 50ms and will pick it up automatically.
